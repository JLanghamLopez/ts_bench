Task: Multivariate Generative Modelling of 24-Hour Cryptocurrency Log-Return Cycles

**1. Task Overview**

This task evaluates a modelâ€™s ability to generate realistic 24-step multivariate log-return sequences for a basket of major cryptocurrencies.
The objective is to capture the distinctive dynamics of digital-asset markets, including high-frequency volatility bursts, cross-asset co-movement patterns, global timezone effects, and rapid sentiment-driven fluctuations.

Task Type: Generative modelling
Trajectory Shape: [24, 10] log-return sequences

**2. Task Description**

Participants must develop a generative model capable of producing realistic 24-hour sequences of hourly log-returns across the following cryptoassets:
ADAUSDT, BNBUSDT, BTCUSDT, DOTUSDT, ETHUSDT, LINKUSDT, LTCUSDT, SOLUSDT, XLMUSDT, XRPUSDT.

Each data sample is a 24-step rolling window of hourly log-returns over ten cryptocurrency pairs.
The model should be able to generate synthetic 24-hour return cycles that reproduce the prominent features of crypto markets, such as sharp jumps following news releases, liquidity-driven regime shifts, rapid inter-asset transmission of shocks, and persistent short-horizon volatility patterns.

Any generative architecture may be used, including diffusion models, GANs, flow-based models, autoregressive generators, or signature-based models, as long as the generated trajectories have the correct [24, 10] structure.

**3. Dataset**

The dataset is derived from hourly price data covering several years of major cryptocurrency trading activity.
Raw prices are transformed into log-returns to normalize scale across assets and highlight proportional changes.

The time series is segmented into windows of length 24 using a stride of 4 hours.
This design captures:
- Full day-long segments reflecting the 24/7 nature of crypto markets,
- Overlapping windows sufficient for dense learning,
- Short-horizon behaviour including volatility clustering and rapid sentiment propagation.

The dataset contains:
- Train set
- Validation set
- Test set

Participants should develop and tune models using the train and validation splits.
The test set will not be provided; final ranking is based solely on performance on the held-out test set using the primary metric (HistogramMetric).

**4. Evaluation**

Generated output is compared to real crypto return sequences using three metrics:

- HistogramMetric
  Measures distributional similarity between real and generated trajectories by comparing empirical histograms of returns.

- CrossCorrelationMetric
  Measures lagged cross-asset correlation behaviour, ensuring the model captures instantaneous and short-lag co-movements across cryptoassets.

- AutoCorrelationMetric
  Assesses the autocorrelation structure of individual crypto return series, reflecting short-range dependencies, volatility dynamics, and hourly return characteristics.

HistogramMetric serves as the primary ranking metric.
CrossCorrelationMetric and AutoCorrelationMetric provide complementary evaluations of multivariate and temporal behaviour.
Evaluation metric code for all three metrics will be provided to participants.

**5. Success Criteria**

A strong submission should:
- Achieve low discrepancy on HistogramMetric when generating 24-hour crypto return paths.
- Preserve realistic inter-asset correlation patterns as assessed by CrossCorrelationMetric.
- Reproduce short-horizon autocorrelation and volatility patterns indicated by AutoCorrelationMetric.
- Generate stable trajectories that avoid artefacts such as constant paths, unrealistic spikes, mode collapse, or excessively smooth behaviour.

**6. Difficulty Level**

Difficulty: Intermediate

Cryptocurrency markets exhibit pronounced non-stationarity, high volatility, rapid structural shifts, and strong cross-asset connectivity.
Accurately modelling these properties within a generative framework requires robust handling of multivariate short-horizon dynamics, non-Gaussian features, and rapidly evolving dependence structures across assets.
